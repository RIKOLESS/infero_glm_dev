# 报告模板 · 风险预警通报

> 场景：面向沿线单位（路政 / 交警 / 应急 / 属地政府）的分级通报。
> 面向：跨部门联动对象。

---

## 骨架

```
【预警通报】<灾种> · <最高级别> · 影响 <省 · 路线 · 桩号>
生效时间：<start UTC+8> — <end UTC+8>
签发单位：<单位>（db 智能体草拟 · 待人工签批）
发布时间：<YYYY-MM-DD HH:mm> UTC+8

一、预警要素
    - 灾种：<暴雨 / 大风 / 大雾 / 雨雪 / 冰冻 / 山洪 / 地震 …>
    - 最高级别：<红 / 橙 / 黄 / 蓝>
    - 关键指标：<例：1h 累计降雨 >50mm；能见度 <100m；风速 >20m/s>

二、影响范围
    - 涉及省份：<省 A · 省 B>
    - 涉及路线：<G5 京昆 · G7611 汶马 …>
    - 重点桩号：<K1250—K1310 段等>
    - 是否叠加隐患点：<桥梁/隧道/边坡 命中情况>

三、防御要点
    · 路政：<沿线巡查频次 · 警示锥预置数量 · 关键桥隧监视>
    · 交警：<关键节点驻守 · 能见度不足时的封控准备 · 车队引导>
    · 应急：<物资 / 力量前置建议 · 装备储备提示>
    · 属地：<积水易涝点、下穿隧道、山区落石区的提前提示>

四、通报与联动
    - 已通报：<对象列表>
    - 大屏跟踪：<shortTerm / analysisReport / threeMap 路径>
    - 复核时间：每 <n> 小时刷新一次数据

五、数据来源
    - 平台接口：<接口名列表>；数据模式：<live/demo>
    - 数据质量：成功 <n> · 空 <n> · 失败 <n · 说明>
    - 生成工具：db 智能体 v<版本>

六、注意事项
    - 本通报为草拟稿，须由值班主任签批后正式发布。
    - 桩号级信息以平台最新数据为准；实际封控 / 管制指令由公安交管部门下达。
```

---

## 填写要点

- **级别**必须与 `publishLevelStats` 一致；不许因语气弱化真实级别。
- **防御要点**四条至少写两条；每条尽量给动作而非空话。
- **通报对象**明确；未通报的写"待通报"。
- **注意事项**保留"草拟 + 待签批"字样，明确 db 的边界。

## 面向沿线单位的语气

- 稳健、克制、不制造恐慌。
- 用"建议 / 请注意 / 可考虑 / 需人工确认"等措辞，避免"必须 / 强制 / 立即执行"。
- 数字要与接口数据一致；不出现无根据的"预计"。
