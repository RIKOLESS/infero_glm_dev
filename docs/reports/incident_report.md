# 报告模板 · 事件通报

> 场景：具体突发事件（山洪、滑坡、桥梁受损、大范围阻断、地震）对内通报。
> 面向：应急指挥中心、路网中心、上级主管部门。

---

## 骨架

```
【事件通报】<事件类型> · <严重程度> · <省 · 路线 · 桩号>
发生时间：<YYYY-MM-DD HH:mm> UTC+8
通报时间：<YYYY-MM-DD HH:mm> UTC+8
草拟：db 智能体 · 待值班主任签批

一、事件概况
    - 事件类型：<暴雨阻断 / 桥梁受损 / 滑坡 / 泥石流 / 地震灾损 / 交通事故 …>
    - 事件位置：<省 · 市 · 县 · 路线 · 桩号（或经纬度）>
    - 严重程度：<I 特别重大 / II 重大 / III 较大 / IV 一般>（暂拟，待现场确认）
    - 起源与背景：<气象诱因 / 地震震例 / 人为事故 …>

二、影响范围
    - 阻断/管控状态：<单向阻断 / 双向阻断 / 限速 / 分流 / 待人工确认>
    - 涉及路段：<主线 · 匝道 · 桥隧>
    - 受影响车流估算：<平台无数据则写"暂无平台侧车流数据">
    - 次生风险：<余震 / 后续降雨 / 山体不稳定 …>

三、响应动作
    - 建议响应级别：<X 级>（触发依据：<接口/字段>）
    - 已启动响应：<等级 · 牵头单位 · 时间>
    - 通报路径：<内部群 → 沿线路政 → 交警 → 属地政府 → 上级主管>
    - 力量部署：<救援点 · 物资库 · 巡查加密>（详见决策建议书）

四、平台联动
    - 大屏：<shortTerm / analysisReport / threeMap 打开状态>
    - 数据刷新：每 <n> 小时复核；下一次预计 <UTC+8>
    - 关联报告：<决策建议书 / 值班简报 编号>

五、数据来源
    - 平台接口：<summary · segmentRisk · resources · earthquake · ...>
    - 数据模式：<live / demo>
    - 数据质量：成功 <n> · 空 <n> · 失败 <n · 说明>
    - 生成时间：<UTC+8>

六、待人工确认事项
    - <阻断/事故的现场核实>
    - <车流量与人员被困情况>
    - <救援力量实际到位情况>
    - <是否需要正式启动 X 级响应>
```

---

## 填写要点

- **事件概况**先给"是什么 · 在哪里 · 多严重"；不铺陈原始数据。
- **响应动作** = 建议 + 已启动，两者要区分清楚，避免让上级误认为已经启动。
- **待人工确认事项**必写；db 不能对现场事实拍板。
- **数据来源**如实标注 live/demo；不遮丑。

## 与其它报告的关系

- 事件通报是"对内首发"，出完后一般紧接 `risk_alert.md` 对外通报。
- 决策类内容独立成 `decision_memo.md`，避免通报里塞过多方案细节。
- 事件闭环后配 `after_action.md`。
