# 报告模板 · 每日总结（日报）

> 场景：每日 20:00 前后出的日终态势 + 值班日结。
> 面向：值班班组、上级主管部门。

---

## 骨架

```
【<省份/全国> 公路气象日报】 <YYYY-MM-DD>
签发：db 智能体草拟 · 值班主任 <姓名> 审核

一、当日总体形势
    - 预警发布：R <n> · O <n> · Y <n> · B <n>；主导灾种：<灾种>
    - 事件计数：阻断 <n> · 事故 <n> · 应急启动 <n>（若无数据请如实说明）
    - 累计降雨 TOP 省份：<省 A n mm · 省 B n mm>
    - 最大能见度事件：<省 · 桩号 · 时段 · 最低能见度 m>

二、重要事件回顾（按时间顺序）
    HH:mm  <事件类型> <位置> <处置动作简述>
    HH:mm  <事件类型> <位置> <处置动作简述>
    ...

三、处置总结
    - 力量投入：路政 <n> 人次 · 交警 <n> 人次 · 应急 <n> 人次（若无数据请如实说明）
    - 物资消耗：<沙袋 n · 融雪剂 n · 警示锥 n>
    - 平台使用：短临大屏跳转 <n> 次 · 分析看板 <n> 次
    - 简报输出：值班简报 <n> · 通报 <n> · 决策建议书 <n>

四、明日展望
    - 预计天气：<灾种 · 级别 · 时段>
    - 重点关注：<路线 · 桩号>
    - 建议动作：<加密巡查 / 力量前置 / 值班加强>

五、数据来源与质量
    - 接口调用汇总：成功 <n> · 空 <n> · 失败 <n>
    - 数据模式：<live / demo>；接口异常项：<列出>
    - 生成时间：<UTC+8>

六、待跟进事项
    - <未闭环事件>
    - <平台侧数据缺口>
    - <流程改进项>
```

---

## 填写要点

- **总体形势**优先给"发布 vs 实际发生"的比对，突出预警命中或未命中。
- **重要事件**按时间线，一行一条；避免大段散文。
- **处置总结**数据能填就填，填不了坦白写"暂无该项统计"。
- **明日展望**保留 3 条以内；不做过远预测。
- **待跟进事项**便于次日交接一目了然。

## 常见坑

- 不要重复今日简报的原文；日报是"聚合叙事"，不是简报堆叠。
- 数据能来自平台的必须来自平台；db 不编造聚合数字。
- 涉及个人（值班姓名、司机姓名）时保留必要字段，避免过度暴露。
