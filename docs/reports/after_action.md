# 报告模板 · 复盘评估（AAR）

> 场景：单次事件闭环、或月度专题复盘。
> 面向：本单位复盘会、上级主管部门归档。

---

## 骨架

```
【复盘评估】<事件名称> · <YYYY-MM-DD>
起止时间：<start> — <end>（UTC+8）
草拟：db 智能体 · 值班主任 <姓名> 审核

一、事件回放
    - 起因：<气象诱因 / 地震 / 人为事件 …>
    - 发展时序：
        HH:mm  预警发布 / 事件发生
        HH:mm  内部通报
        HH:mm  力量到位
        HH:mm  处置见效
        HH:mm  事件闭环
    - 关键节点决策：<决策 · 决策依据 · 决策者>

二、成效指标
    - 预警提前量：<n 分钟>（预警发布 → 事件发生）
    - 响应时长：<n 分钟>（通报 → 力量到位）
    - 阻断/管制时长：<n 分钟>
    - 人员/物资投入：<n>
    - 是否发生人员伤亡：<无 / 有：简述>
    - 事件影响面：<涉及路线/桩号/车流估算>

三、亮点
    1. <预警准确 / 通报及时 / 力量匹配 / 平台联动顺畅 …>
    2. ...

四、不足与原因分析
    1. <预警偏晚 / 数据接口空 / 通报断点 / 物资匹配偏差 …>
       原因：<接口延迟 / 岗位切换 / 经验不足 …>
    2. ...

五、改进建议
    1. <数据层：某接口延迟改善方案>
    2. <流程层：通报路径优化建议>
    3. <平台层：大屏 / 图层新增需求>
    4. <培训层：值班培训 / 演练建议>

六、经验教训（供归档）
    - 短临降雨触发规则：<可否调整阈值>
    - 力量部署经验值：<沙袋/融雪剂/警示锥 用量校准>
    - 通报模板经验：<语言风格 / 格式改进>

七、数据来源与质量
    - 平台接口调用：<清单>
    - 数据模式：<live / demo>
    - 数据质量：成功 <n> · 空 <n> · 失败 <n · 说明>
    - 生成时间：<UTC+8>
```

---

## 填写要点

- **事件回放**用时间线，不用长散文。
- **成效指标**能有数字就有数字；无数字就写"暂无该项统计"。
- **不足与原因**要成对出现（"哪里不足 + 为什么"）；忌只写现象不写归因。
- **改进建议**分层：数据 / 流程 / 平台 / 培训；便于对号入座落实。
- **经验教训**尽量沉淀成可复用的规则，反哺 playbook 与经验值表。
