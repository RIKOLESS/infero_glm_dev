# 报告模板 · 值班简报

> 场景：日常值班交接 / 定时简报 / 常态化态势播报。
> 面向：值班主任、值班团队。

---

## 骨架

```
【<省份/全国> 公路气象值班简报 · <YYYY-MM-DD HH:mm> UTC+8】

一、总体形势
    - 预警概况：R <n> / O <n> / Y <n> / B <n>；主导灾种：<灾种>
    - 影响范围：<省 · 路线 · 桩号，若无桩号级则说明>
    - 与上一时段对比：<持平 / 增强 / 减弱>

二、重点关注路段（TOP N）
    1. <路线 · 桩号> · <降雨/风速/能见度> · <风险标签>
    2. ...
    （每条一行，包含所属路线、桩号或路段名、关键指标、是否命中风险区/隐患点）

三、平台联动
    - 已打开大屏：<shortTerm / analysisReport / threeMap / rainfall>
    - 平台侧任务：<例：路网中心已发布 III 级预警 / 上一时段应急指挥中心已通报>

四、建议动作
    1. <加密巡查 / 力量前置 / 物资预置 …>
    2. <通报对象 · 时限>
    3. <大屏 / 简报 / 通报 发布计划>

五、数据来源与质量
    - 调用接口：<summary · shortTerm · segmentRisk · resources · earthquake · defense>
    - 成功：<n>；空数据：<n>；失败：<n · 说明>
    - 数据模式：<live / demo>；生成时间：<UTC+8>

六、签发
    - db 智能体草拟 · 待值班主任审核
```

---

## 填写要点

- **总体形势**先给结论，不铺陈原始数据。
- **TOP N** 建议 5–10 条；数据不足时如实标"本轮无桩号级明细"。
- **建议动作**每条要"谁 · 做什么 · 什么时候前"三要素完整。
- **数据来源**必写；成功/空/失败清单要具体。
- **签发**保持"待审核"字样，明确 db 只做草拟。

## 常用变量约定

| 变量 | 来源 |
|---|---|
| `<省份/全国>` | 用户输入 或 默认全国 |
| `<灾种>` | 从 `defense.publishLevelStats` 或 `summary` 里最大占比一类灾种 |
| `<TOP N>` | 从 `shortTermRank / affectedTop10 / highImpactSegments` 归并 |
| `<数据模式>` | `report.mode`（`live / demo / auto`） |
| `<失败接口>` | `report.source_summary` 中 `status='failed'` 的 endpoint 列表 |
