# 报告模板 · 决策建议书

> 场景：为具体决策（力量预置 / 物资调拨 / 救援调派 / 响应级别启动）撰写的方案陈述。
> 面向：决策者（值班主任 / 应急指挥中心 / 上级领导）。

---

## 骨架

```
【决策建议书】<方案主题> · <YYYY-MM-DD HH:mm> UTC+8
草拟：db 智能体 · 待值班主任签批

一、背景
    - 触发事件：<简述气象/地震/事故触发条件>
    - 数据支撑：<关键接口 · 关键字段 · 数据模式 live/demo>
    - 决策时限：<必须在 <UTC+8> 前答复>

二、方案概要
    - 建议动作：<一句话摘要>
    - 预期效果：<一句话陈述>
    - 主要成本：<人 · 物 · 时间>
    - 风险与不确定性：<列出关键假设>

三、方案详情
    3.1 力量与物资部署（引用 vehicle_rescue / material_dispatch playbook）
        - 建议救援点：<点名 · 距离 ~X km 直线 · 车辆数 · 电话>
        - 备选救援点：<点名 …>
        - 建议物资库：<库名 · 距离 · 匹配物资>
        - 建议物资数量：<品名 数量>（经验值，需现场核对）

    3.2 人力与岗位调整（引用 force_prepositioning playbook）
        - 巡查加密：<路段 · 频次 · 时段>
        - 关键节点驻守：<桥梁 / 隧道 / 易积水区>
        - 值班加强：<AB 岗 / 24h 主任在岗>

    3.3 平台联动
        - 大屏跳转：<shortTerm / analysisReport / threeMap>
        - 数据刷新计划：每 <n> 小时复核

四、备选方案
    - 方案 A（推荐）：<摘要 · 优缺点>
    - 方案 B（备用）：<摘要 · 优缺点>
    - 方案 C（保底）：<摘要 · 优缺点>

五、待人工确认事项
    - <救援点电话是否可通>
    - <物资库存明细是否符合>
    - <响应级别是否正式启动>
    - <跨省协调是否已上报>

六、数据来源
    - 平台接口：<summary · segmentRisk · resources · earthquake · ...>
    - 数据模式：<live / demo>
    - 数据质量：成功 <n> · 空 <n> · 失败 <n · 说明>
    - 生成时间：<UTC+8>

七、边界声明
    - 本建议由 db 智能体草拟，未涉及正式指令下达。
    - 直线距离为经纬度粗算，非路网导航时间。
    - 决策以值班主任 / 指挥部门签批为准。
```

---

## 填写要点

- **背景**不多于 5 行；直接给出触发条件与决策时限。
- **方案概要**先给"建议 · 效果 · 成本 · 风险"，让决策者 30 秒内做判断。
- **方案详情**引用对应 playbook；不重复长文字。
- **备选方案**必须给出至少 2 个；避免"只有一个选项"。
- **待人工确认事项**必写；提醒决策者哪些不能靠算法。
- **边界声明**保留 3 条；明确 db 不是指挥官。

## 语气

- 稳健、克制、避免"必须 / 唯一"等强断句。
- 使用"建议 / 可考虑 / 需人工确认"等措辞。
- 每个数字都要能追溯到平台接口 或 经验值备注。
