## 通用 5 阶段结构

每份 playbook 都按 `预报 → 研判 → 决策 → 处置 → 评估` 组织：

1. **预报（forecast）**：拿哪些数据、看哪些指标，明确要预警什么。
2. **研判（analyze）**：数据交叉、影响范围、风险等级判定。
3. **决策（decide）**：给谁看、走什么响应级别、可选方案。
4. **处置（act）**：具体动作清单，包括平台操作、通知对象、简报输出。
5. **评估（review）**：事后复盘要素、成效指标、可归档要点。

## 使用约定

- SOP 是骨架，不是脚本；文本、语气、结论要结合当前数据实际情况。
- 遇到未列入的场景，就走 `defense_advisory.md`（通用防御提示）流程，同时向用户说明"该场景暂无专用 SOP"。
- 每份 playbook 末尾都指定推荐的 report 模板，处置完随手 `kb.read('reports/xxx')` 出简报。
