# playbook · 应急装备物资储备调配

> 适用触发词：物资、装备、储备、调拨、储备库、库存、防汛物资、融雪剂、沙袋。
> 场景对齐：比赛场景 4（应急装备物资储备调配）。
> 推荐报告模板：`reports/decision_memo.md`。

---

## 一、预报（forecast）

**目标**：面向即将发生 / 已发生的灾情，给出调拨建议方案（哪库出、多少、送到哪里）。

数据源与调用：

- `await window.weatherPlatform.resources({ province })`：拿 `materialReserves` + `vehicleRescuePoints` + `riskSegments`。
- `await window.weatherPlatform.summary({ province })`：拿气象态势判定物资类别。
- `await window.weatherPlatform.segmentRisk({ province })`：叠加隐患点，判断哪些路段"需物资前置"。

关键字段：

- `materialReserves`：`name / province / city / lon / lat / stockpile`（stockpile 为库存明细，字段名视平台而定，adapter 会保留原始 JSON）。
- 事件位置：由用户提供或由 `affectedTop10 / riskSegments` 推断。

---

## 二、研判（analyze）

按四步：

1. **需求类型识别**：
   - 暴雨/山洪：沙袋、排水泵、警示锥、发电机。
   - 大风/台风：临时导向、警戒带、加固支柱。
   - 冰雪：融雪剂、防滑链、清雪车、破冰器材。
   - 地震/地灾：破碎机、切割机、生命探测仪。
2. **候选库匹配**：
   - 从 `materialReserves` 筛所在省 + 相邻省的库。
   - 按 `stockpile` 里是否含目标物资类别筛（若数据字段不明确，就说明"库存明细需人工到平台核对"）。
   - 直线距离 haversine 粗排序，取 2–3 个候选。
3. **调拨数量估算**：
   - 参考基线（可写进 SOP 附表）：
     - 沙袋：每 100m 阻断预置 ~200 个。
     - 融雪剂：每千米路面 100–200 kg（视降雪强度）。
     - 警示锥：每处封控点 20 个。
   - 明确"上述为经验值，非平台数据；实际数量由值班主任按经验决定"。
4. **路线可达性**：
   - 结合当前气象预警，判断调拨车辆沿线是否需要防护。
   - 如涉及跨省，需提示"跨省调拨须走上级协调"。

---

## 三、决策（decide）

输出方案骨架：

```
物资调拨建议 v1（草案）
- 事件锚点：<路线 · 桩号 · 省份>
- 需求类型：<灾种> → <物资类别>
- 首选库：<库名>（距离 ~X km 直线，库存 <明细/需人工核实>）
- 次选库：<库名>（距离 ~Y km）
- 建议数量：<品名 数量>（经验值，需值班主任确认）
- 调拨车辆：建议由 <车辆救援点> 承担运输
- 沿线风险：<是否需防护>
- 责任对接：<单位/人员>（须人工确认）
```

明确边界：

- db 不下达调拨指令；不代替库管员盘点真实库存。
- 数量为经验估算，需要人工按现场情况调整。

---

## 四、处置（act）

db 侧动作：

1. `renderStatus(bundle)`：把候选库 + 需求物资列在 Canvas。
2. 出 `reports/decision_memo.md`：条目化列出调拨方案。
3. 打开 3D 大屏 `openBigScreen('threeMap')` 便于值班员空间核对。
4. 附上原始数据源与数据模式（live/demo）。

后续动作：

- 与 `vehicle_rescue.md` 联动：调拨车辆与救援车辆是否复用同一救援点。
- 与 `force_prepositioning.md` 联动：物资预置往往是力量预置的一部分。

---

## 五、评估（review）

事件闭环后：

- 记录实际调拨与建议方案偏差（品类 / 数量 / 库源）。
- 记录调拨到位时长、消耗量、剩余量。
- 反哺经验值表（沙袋/融雪剂/警示锥的用量基线）。
- 用 `reports/after_action.md` 出复盘小节。

---

## 附：常见坑与提醒

- **stockpile 字段不稳定**：平台不同版本字段可能是 `stockpile / materials / detail`；解析不到就明说需人工核对。
- **调拨路径**：db 只算直线距离，不算路网时间；正式方案由调度指挥台确认。
- **跨省调拨**：涉及不同省份指挥体系时，务必提示走上级协调。
- **不重复劝退**：如平台已有正在调拨的物资，db 应识别并避免重复建议（若平台无此字段，就在末尾提示"是否已有在途调拨请人工确认"）。
