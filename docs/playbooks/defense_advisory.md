# playbook · 公路气象预警防御提示

> 适用触发词：防御提示、通报、预警发布、值班通知、面向公众、防御指引。
> 场景对齐：比赛场景 7（公路气象预警防御提示）；也是**未列入其他场景时的通用兜底流程**。
> 推荐报告模板：`reports/risk_alert.md`（风险预警通报）+ `reports/duty_brief.md`。

---

## 一、预报（forecast）

**目标**：把平台预警数据转成可直接对外发布的"防御提示文本"，避免值班员手写通报。

数据源与调用：

- `await window.weatherPlatform.defense({ province })`：按四种层级（`typhoon / rainstorm / snowstorm / freezingRain` 或平台自定义）返回预警级别 × 数量。
- `await window.weatherPlatform.summary({ province })`：拿总预警数 + TOP 影响路段。
- `await window.weatherPlatform.segmentRisk({ province })`：叠加风险路段作为具体到桩号的说明。

关键字段：

- `publishLevelStats`：R/O/Y/B 数量，按预警类型分组。
- `alertSearch / alertRegionStats`：明细预警条目（如需 quote 到通报里）。

---

## 二、研判（analyze）

按四步：

1. **预警最高等级**：从 `publishLevelStats` 汇总，取本轮最高级别（R > O > Y > B）。
2. **主导灾种**：判断本轮以哪个灾种为主（暴雨 / 大风 / 大雾 / 雨雪 / 冰冻）。
3. **地理范围**：影响省份、重点路线，尽量精准到桩号（若接口不返回桩号就说明"以路线级别为准"）。
4. **面向对象**：
   - 面向值班主任：内部简报，条目化 + 分级建议。
   - 面向沿线单位：路政 / 交警 / 属地政府 / 应急管理部门。
   - 面向公众：出行提示（简明，不含内部术语）。

---

## 三、决策（decide）

针对三类受众给三段文本骨架：

### A. 面向值班主任（内部简报，走 `reports/duty_brief.md`）

```
【<省份/全国> 气象预警值班简报 · <时间>】
一、总体形势：<X 项红色 / Y 项橙色 / Z 项黄色>，主导灾种：<灾种>
二、重点关注路段：<TOP N，含桩号>
三、建议动作：
  1. <加密巡查/预置力量/大屏跟踪 …>
  2. <通报对象>
四、数据来源：WeatherAdapter · luwang（<模式：live/demo>），成功 <n> · 空 <n> · 失败 <n>
```

### B. 面向沿线单位（分级通报，走 `reports/risk_alert.md`）

```
【预警通报】<灾种> · <最高级别> · 影响 <省 · 路线 · 桩号>
生效时间：<start> — <end>
主要影响：<桩号级降雨/风速/能见度>
防御要点：
  · 路政：<沿线巡查频次、警示锥预置>
  · 交警：<关键节点驻守、能见度不足时的封控准备>
  · 应急：<物资 / 力量前置建议>
签发：<单位 · 时间>
```

### C. 面向公众（出行提示）

```
【出行提示】<灾种> · <省份>
未来 <时间窗口>，<省份/路线> 将出现 <灾种简述>。建议：
  1. <减速慢行 / 保持车距 / 避开陡坡下坡 / 尽量绕行>
  2. <关注实时路况、必要时短驻服务区>
  3. <途经 <重点路段> 时特别注意>
如遇险情请拨打 <公开求助电话>。
```

**注意**：面向公众文本必须避免内部术语、桩号、内部电话；简明、可读、可复制。

---

## 四、处置（act）

db 侧动作：

1. `renderStatus(bundle)`：Canvas 展示当前预警分级与主导灾种。
2. `renderReport(report)` + `openReport / exportReport`：把三段文本 + 数据来源整合成一份完整报告。
3. `openBigScreen('shortTerm')` 或 `analysisReport`：值班员可即时切换到大屏核对。
4. 复制文本按钮：让值班员一键把内部/外部通报复制走。

---

## 五、评估（review）

事件结束后：

- 记录本轮预警 → 实际发生比对（预报准确率、公众反馈）。
- 记录通报是否被上级 / 沿线单位采信（有无反馈）。
- 记录接口数据的完整度与延迟。
- 用 `reports/after_action.md` 出复盘，重点写"文本可读性 + 数据完备度"。

---

## 附：通用兜底流程

如果任务不属于其他 6 个 playbook 覆盖场景：

1. 用 `summary` 拿总体形势。
2. 判断灾种和最高级别。
3. 按本 SOP 给出内部简报 + 分级通报 + 公众提示三段文本。
4. 结尾附一句"该场景暂无专用 SOP，走通用防御提示流程"以便值班员知晓。
